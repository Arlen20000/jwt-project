# 🔐 JWT Лаборатория — Задание №27

Интерактивная веб-лаборатория по созданию и верификации JWT токенов вручную, без использования готовых JWT-библиотек.

## 📋 О проекте

Лабораторная работа по дисциплине **Криптография / Кибербезопасность**, Курс 2.

Реализовано:
- Создание и верификация JWT через **HMAC-SHA256** (HS256)
- Создание и верификация JWT через **RSA-SHA256** (RS256)
- Демонстрация атаки **alg:none**
- Декодирование любого JWT токена

## 🔑 Как работает JWT

JWT состоит из трёх частей разделённых точкой:

```
header.payload.signature
```

- **Header** — алгоритм подписи (HS256 / RS256)
- **Payload** — данные пользователя (sub, role, exp...)
- **Signature** — криптографическая подпись, защищает от подделки

## ⚔️ Атака alg:none

Уязвимые реализации JWT принимают алгоритм из заголовка токена. Атакующий меняет `alg` на `none`, подменяет payload (например `role: admin`) и убирает подпись — уязвимый сервер принимает такой токен без проверки.

**Защита:** всегда жёстко указывать разрешённые алгоритмы при верификации.

## 📡 API эндпоинты

|---|---|---|
| `/api/hmac/create` | POST | Создать HMAC токен |
| `/api/hmac/verify` | POST | Верифицировать HMAC токен |
| `/api/rsa/keys` | GET | Получить RSA ключи |
| `/api/rsa/create` | POST | Создать RSA токен |
| `/api/rsa/verify` | POST | Верифицировать RSA токен |

| `/api/decode` | POST | Декодировать токен |
