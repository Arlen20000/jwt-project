🔍 Декодирование токена (без верификации)

JWT Base64URL декодируется без ключа — это не верификация. Подпись при этом НЕ проверяется.

Что такое JWT?
JSON Web Token (RFC 7519) — компактный способ передачи информации между сторонами в виде JSON-объекта с цифровой подписью. Токен состоит из трёх частей, разделённых точкой: Header.Payload.Signature
Структура токена
Header — алгоритм подписи и тип токена: {"alg":"HS256","typ":"JWT"}
Payload — claims (утверждения): sub, name, role, iat, exp
Signature — HMAC-SHA256(base64url(header) + "." + base64url(payload), secret)

Все части кодируются в Base64URL (без паддинга =)

⚠️ Атака alg:none (CVE-2015-9235 и аналоги)
Уязвимые библиотеки (например, node-jsonwebtoken до 4.2.2) принимали любой алгоритм, указанный в заголовке токена, включая "none". При alg:none подпись не требуется. Атакующий мог: 1) взять любой токен, 2) изменить payload (повысить права), 3) поставить alg:none, 4) убрать подпись. Уязвимый сервер принимал такой токен как валидный.
✅ Применение JWT в веб-аутентификации
1. Пользователь логинится → сервер создаёт JWT с ролями и временем жизни (exp)
2. Клиент хранит токен (localStorage или httpOnly cookie)
3. При каждом запросе: Authorization: Bearer <token>
4. Сервер верифицирует подпись и проверяет exp → нет обращений к БД
5. Refresh token используется для получения нового access token

Преимущество: stateless — сервер не хранит сессии, горизонтальное масштабирование.
